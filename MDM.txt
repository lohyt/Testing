✅ Device Owner (DO)

You’ll see something like:

Device Owner:
  admin=ComponentInfo{com.company.mdm/.AdminReceiver}

👉 Meaning:

Full device control
Global policies (camera, etc.)

🧑‍💼 Profile Owner (PO)

You’ll see:

Profile Owner (User 10):
  admin=ComponentInfo{com.company.mdm/.AdminReceiver}

👉 Meaning:

Only work profile controlled
Personal side untouched

❌ No Owner (Weak / Not Managed)
(no device owner)
(no profile owner)

👉 Meaning:

Not properly enrolled

🧠 High-Level Flow (What Happens Internally)
When camera is blocked:


MDM calls:

Framework updates:


DevicePolicyManagerService




When any app tries camera:


CameraService checks policy


Denies access





🧪 PART 1 — Trace MDM Setting Camera Policy
🎯 Attach Frida to MDM app
frida -U -n <mdm_package>

🧠 Hook this method
Java.perform(function () {    var dpm = Java.use("android.app.admin.DevicePolicyManager");    dpm.setCameraDisabled.overload(        'android.content.ComponentName',        'boolean'    ).implementation = function (admin, disabled) {        console.log("[+] setCameraDisabled called: " + disabled);        return this.setCameraDisabled(admin, disabled);    };});

🔍 What you’ll see


true → camera being blocked


false → camera being enabled


👉 This confirms:

MDM is using proper API (or not)


🧪 PART 2 — Trace Camera Access Attempts
🎯 Attach to any app (camera / browser)
frida -U -f <app_package> -l hook.js --no-pause

🧠 Hook Camera APIs
Java.perform(function () {    var CameraManager = Java.use("android.hardware.camera2.CameraManager");    CameraManager.openCamera.overload(        'java.lang.String',        'android.hardware.camera2.CameraDevice$StateCallback',        'android.os.Handler'    ).implementation = function (id, cb, handler) {        console.log("[+] openCamera called: " + id);        try {            return this.openCamera(id, cb, handler);        } catch (e) {            console.log("[!] Blocked: " + e);            throw e;        }    };});

🔍 Expected Output
When camera is blocked:
[+] openCamera called: 0[!] Blocked: CameraAccessException: disabled by policy
👉 This proves:

Blocking happens at framework level
